
# Release 1.4.23 (2026-05-09)

## Bug fixes

- **pass boolean to ci-security reusable workflow inputs**
The security reusable workflow declares run-standards and run-security as type: boolean. The caller was passing values via the || operator which produces strings, causing the reusable workflow to fail silently. Use the != 'false' pattern to produce actual booleans. Ref wphillipmoore/standard-tooling#632
